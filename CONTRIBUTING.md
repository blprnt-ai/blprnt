# Contributing to blprnt

Focused pull requests are welcome.

## Guidelines

- Keep changes scoped. Small, reviewable PRs beat heroic messes.
- Explain why the change is needed and call out any user-visible impact in the PR description.
- Match the existing stack, structure, and local code patterns instead of inventing a new religion.
- If your change affects behavior, include enough context for reviewers to understand what changed.

## Legal and Process

- No CLA is required.
- No DCO sign-off is required.
- By contributing, you agree that your contribution is provided under the repository's license terms. Read `LICENSE` before opening a pull request.
