# Accessibility Checklist

## Keyboard & Focus
- All interactive elements reachable via keyboard.
- Visible focus styles on all focusable elements.
- Logical tab order matching visual layout.

## Semantics & Labels
- Use semantic HTML elements when possible.
- Inputs have associated labels.
- Buttons and links have descriptive text.

## Contrast & Readability
- Text contrast meets WCAG AA (4.5:1 normal, 3:1 large).
- Placeholder text is not the only label.
- Avoid low-contrast disabled states for critical info.

## ARIA
- Use ARIA only when semantic HTML is insufficient.
- Ensure ARIA labels are unique and descriptive.
- Avoid duplicate IDs or conflicting roles.
