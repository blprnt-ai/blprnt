# Frontend Design Principles

## Visual Hierarchy
- Use size, weight, and spacing to guide attention.
- Primary actions should be most prominent.
- Reduce competing emphasis in secondary elements.

## Spacing & Alignment
- Use consistent spacing scale and alignment.
- Prefer predictable grids and aligned edges.
- Avoid arbitrary spacing unless justified.

## Typography
- Limit to established type scales and weights.
- Use clear heading levels for structure.
- Maintain readable line lengths (50–80 chars).

## Color & Contrast
- Use existing color tokens for states.
- Ensure sufficient contrast for text and controls.
- Avoid color as the only signal for state.

## States & Feedback
- Define hover, focus, active, and disabled states.
- Provide immediate feedback for user actions.
- Ensure transitions are subtle and consistent.
