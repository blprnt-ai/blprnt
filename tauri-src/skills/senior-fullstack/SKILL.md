# Senior Fullstack

Senior-level fullstack guidance for shaping robust applications with modern tools and best practices.

## Core Capabilities

### 1. Fullstack architecture guidance

Guidance for structuring full applications across frontend, backend, data, and infrastructure layers.

**Features:**
- End-to-end architecture thinking
- Practical best practices
- Adaptable patterns
- Attention to maintainability and scale

### 2. Project structure and delivery

Support for organizing projects so teams can move quickly without creating unnecessary complexity.

**Features:**
- Clear layering and boundaries
- Sustainable conventions
- Delivery-focused trade-offs
- Maintainable defaults

### 3. Code quality and review mindset

Guidance for evaluating implementation quality and raising the bar on reliability.

**Features:**
- Code quality heuristics
- Performance awareness
- Security considerations
- Production-ready patterns

## How to Use This Skill

Use this skill as a senior fullstack reference during design and implementation work.

It is most useful when you need to:
- Choose a stack or architecture direction
- Structure a frontend and backend coherently
- Apply patterns from the reference documents
- Review code for maintainability, performance, and security
- Turn product requirements into pragmatic implementation decisions

## Development Approach

### 1. Clarify product and system requirements

Define the user-facing outcomes, core flows, integrations, and operational constraints before locking in structure.

### 2. Choose pragmatic architecture

Match the stack and patterns to team size, product complexity, performance needs, and expected growth.

### 3. Implement with clear boundaries

Use the reference materials to keep responsibilities separated across UI, services, data access, and infrastructure.

### 4. Review for quality continuously

Check work against maintainability, correctness, observability, security, and performance expectations.

## Best Practices Summary

### Code Quality
- Follow established patterns
- Write comprehensive tests
- Document decisions
- Review regularly

### Performance
- Measure before optimizing
- Use appropriate caching
- Optimize critical paths
- Monitor in production

### Security
- Validate all inputs
- Use parameterized queries
- Implement proper authentication
- Keep dependencies updated

### Maintainability
- Write clear code
- Use consistent naming
- Add helpful comments
- Keep it simple
